重拾Java（三）对象与类

发表于 2019-04-18 | 更新于 2019-05-18 | 分类于 Java

面向对象程序设计概述

  面向对象程序设计（简称 OOP ）是当今主流的程序设计类型。
  面向对象的程序是由杜喜爱那个组成的，每个对象包含对用户公开的特定功能部分和隐藏的实现部分。程序中的很多对象来自标准库，还有一些是自定义的。从根本上来说，只要对象能够满足要求，就不必关心其功能的具体实现过程。
  对于一些规模较小的问题，将其分解为过程的开发模式比较理想。而面向对象更加适用于解决规模较大的问题。要想实现一个简单的 Web浏览器 可能需要大概2000个过程，这些过程可能需要对一组全局数据进行操作。采用面向对象的设计风格，可能只需要大约100个类，每个类平均包含20个方法。

类

  类（class）是构造对象的模板或蓝图，由类构造对象&  的过程称为创建类的实例。
封装是将数据和行为组合在一个包中，并对对象的使用者隐藏了数据的实现方式。数据称为实例域，操纵数据的过程称为方法。实现封装的关键在于绝对不能让类中的方法直接访问其他类的实例域。这是提高重用性和可靠性的关键。
  定义类的方法：修饰符 返回值类型 方法名 ( 参数列表) { 方法体语句； }

类之间的关系

  依赖：应该尽可能的将相互依赖的类减至最少（耦合度最小）。
  聚合：类A的对象包含类B的对象 。
  继承：子类继承父类，拥有父类方法的同时还可以定义自己的方法。

用户自定义类

  • 在一个源文件中，只能有一个公有类，但可以有任意数目的非公有类。
  • this代表本类，为隐式参数

构造器

  • 构造器是一种特殊的方法，用来构造并初始化对象。
  • 构造器与类同名。
  • 每个类可以有一个以上的构造器。
  • 构造器可以有0个，1个或多个参数。
  • 构造器没有返回值。
  • 修饰符一般为public。
  • 构造器总是伴随着new操作一起调用。

封装的优点

一般需要获得或设置实例域的值，需要提供三项内容：

  • 一个私有的数据域；
  • 一个公有的域访问器方法；
  • 一个公有的域更改器方法；

可以改变内部实现，除了该类的方法之外，不会影响其他代码。

final实例域

构建对象是必须初始化这样的域，并且在后面的操作中，不能够再对它进行修改。

静态域和静态方法

静态域

如果把域定义为static，它属于类，而不属于任何独立的对象,即每个实例都将共享一个域。

静态方法

静态方法是一种不能向对象实施操作的方法，建议使用类名，而不是对象来调用静态方法。
使用静态方法的情况有两种，一种是一个方法不需要访问对象状态，其所需参数都是通过显式参数提供。另一种是一个方法只需要访问类的静态域。

对象构造

重载

  • 如果多个方法有相同的名字、不同的参数，便产生了重载。
  • 编译器必须挑选出具体执行哪个方法，他通过用各个方法给出的参数类型与特定方法调用所使用的值类型进行匹配来挑选出相应的方法。
  • 如果编译器找不到匹配的参数，或者找出多个可能的匹配，就会产生编译时错误，这个过程被称为重载解析。
  • Java允许重载任何方法，而不只是构造器方法。
  • 如果在构造器中没有显式的给域赋值，那么就会被自动的赋为默认值：数值为0，布尔型为false，对象引用为null。

调用另一个构造器

当调用new Employee(6000)时，Employee(double)构造器将调用Employee(String, double)构造器，采用这种方式使用this关键字非常有用，这样对公共的构造器代码部分之便写一次即可。

包

Java允许使用包将类组织起来。借助于包可以方便地组织自己的代码，并将自己的代码和别人提供的代码库分开管理。
使用包的主要原因是确保类名的唯一性。为了确保包名的唯一性，Sun公司建议将公司的因特网域名以逆序的形式作为报名，并且对于不同的项目使用不同的子包。

类设计技巧

  • 一定要保证数据私有
  • 一定要对数据初始化
  • 不要在类中使用过多的基本类型
  • 不是所有的域都需要独立的域访问器和域更改器
  • 将职责过多的类进行分解
  • 类名和方法名要能够体现他们的职责
